Variables and Operators

Variables are containers for storing data values. Operators are used to perform operations on variables and values.

1.1 Basic Data Types

  • Text Type: str
  • Numeric Types: int, float
  • Boolean Type: bool

1.2 Type Conversion

In order to specify a type on to a variable, we can use the following:

  • The int function takes any value and converts it to an integer.
    • Example: int(123.22)
  • The float function converts integers and strings to floating-point numbers.
    • Example: float("123")
  • The str function converts to type string.
    • Example: str(123)

1.3 Variables and Assignment

In Python, a variable is just a name

Assignment: variables provide a way to associate names with objects

  • pi = 3
  • radius = 11
  • area = pi * (radius ** 2)

1.4 Operators

Definition: Special symbols that represent computations like addition (+), subtraction (-), multiplication (*) and division (/).

The values the operatoruses are called operands

Arithmetic Operators

Operator Description Example Shortcut
+ Addition Add values on either side of the operator. a + b = 30 x += y
- Subtraction Subtracts right hand operand from left hand operand. a - b = -10 x -= y
* Multiplication Multiplies values on either side of the operator. a * b = 200 x *= y
/ Division Divides left hand operand by right hand operand. b / a = 2.0 x /= y
% Modulus Divides left hand operand by right hand operand and returns remainder. b % a = 1 x %= y
** Exponent Performs exponential (power) calculation on operators. a**b x **= y
// Floor Division The division of operands where the result is the quotient in which the digits after the decimal point are removed. But if one of the operands is negative, the result is floored. 9 // 2 = 4 x //= y

Comparison Operators

Operator Description Example
== If the values of two operands are equal, then the condition becomes true. a == b
!= If values of two operands are not equal, then condition becomes true. a != b
> If the value of left operand is greater than the value of right operand, then condition becomes true. a > b
< If the value of left operand is less than the value of right operand, then condition becomes true. a < b
>= If the value of left operand is greater than or equal to the value of right operand, then condition becomes true. a >= b
<= If the value of left operand is less than or equal to the value of right operand, then condition becomes true. a <= b

When more than one operator appears in an expression, the order of evaluation depends on the rules of precedence, from highest to lowest.

Functions

A function is a block of code which only runs when it is called.

You can pass data, known as parameters, into a function.

A function can return data as a result.

2.1 Functions in Programming

In Python, a function is defined using the def keyword.

Example: def my_function():

2.2 Function Calls

To call a function, use the function name followed by parenthesis.

Example: my_function()

2.3 Parameters and Arguments

Information can be passed into functions as arguments.

Arguments are specified after the function name, inside the parentheses. You can add as many arguments as you want, just separate them with a comma.

Example: def my_function(parameter)

2.4 Built-in Functions

Python has a set of built-in functions.

Example: abs(), all()

Functions and Modules

3.1 Python Functions

  • Keyword def marks the start of function header.
  • A function name to uniquely identify it.
  • Arguments through which we pass values to a function.
  • A colon (:) to mark the end of functino header.
  • Optional documentation string (docstring) to describe what the function does.
  • One or more valid python statements that make up the function body.
  • An optional return statement to return a value from the function.

3.2 The Return Statement

  • Format: return expression
  • Any statement after return are ignored.
  • Pass a value from the function to the caller.

3.3 Flow of Execution

The order in which statements are executed.

  • Execution always begins at the first statement of the program.
  • Statements are executed one at a time, in order from top to bottom.
  • Function definitions do not alter the flow of execution of the program - statements inside the function are not executed until the function is called.

3.4 Python Functions vs Methods

  • Python function is part of the Python script file whereas Methods are defined inside a class definition.
  • We can call a function directly if it’s in the same module or import a module and then call the function directly. To call a method, we need a class or an object of the class.
  • Python function can access all the global variables whereas Python class methods can access global variables, class attributes and functions.

Creating and Importing Modules

4.1 Global and Local Variables

  • Global Variables: defined and declared outside a function. We need to use them inside a function.
  • Local Variables: visibility only inside the function definition. They are available only while the function is executing.

4.2 Nested Function

Definition: A function defined inside another function.

4.3 Modules

  • Modules: library of functions and variables.
  • To access a module, we need to import them: import <module name>
  • To access functions: <Module name>.<function name>(<arguments>)
  • To access variables in a module: <Module name>.<variable name>
  • To access only a selected function in a module: from <module name> import <function name>
  • To everything from a module: from <module name> import *

Strings

Strings in Python are surrounded by either single quotation marks, or double quotation marks.

5.1 Strings are indexed

Index: always starts at 0

Example: 'mouse'

5.2 String Slicing

Slice string with [start:stop:step]

5.3 Strings are immutable

You cannot change elements in a string.

But strings can bound to a new object.

5.4 String Operator

  • Operator: in
  • To test if str1 is a substring of str2
  • Evaluates to a bool

5.5 Strings Methods

  • Python has a set of built-in methods that we can use on strings.
  • All string methods returns new values. They do not change the original string.
  • Format: <string name>.<method name>(x,y,...)

Specification and Testing

Understand software specification, precondition, program bugs, testing and debugging.

6.1 Precondition

Definition: a condition that must always be true just prior to the execution of some section of code or before an operation in a formal specification.

6.2 Basic Terminology

  • Bug: an error in a program.
  • Debugging: The process of finding bugs and remove them.
  • Testing: the process of analyzing the running a program.
  • Test case: a set of input values, together with the expected output.

6.3 Debugging

  • Programming errors are called bugs.
  • The process of tracking bugs and correcting them is called debugging.
  • Three kinds of errors kinds of errors can occur in a program:
    • Syntax errors
    • Runtime errors
    • Semantic error

Conditionals and Control Flow

Understand and use Boolean expressions, if, elif, else, nested if-statements, and program control flow.

7.1 If-Statements

If <boolean-expression> is true, then execute all of the statements indented directly underneath.

7.2 If-Else-Statements

If <boolean-expression> is true, then execute all of the statements indented under if; otherwise, execute the statements indented under else.

7.3 If-Elif-Else_Statements

If <boolean-expression> is true, then execute all of the statements indented under if; otherwise, execute the statements indented under elif and so forth, but lastly, otherwise, execute the statements under else.

7.4 Nested-If

Nested-If simply means there is another If-Statement inside an If-Statement.

Memory in Python

Understand the use of global memory, heap space, function call stack, and memory allocation.

8.1 Global Space and Heap Space

Global Space is:

  • What you "start with."
  • Used to store global variables.
  • Last until you quit Python.

Heap Space is:

  • Where "folders" are stored.
  • Have to access indirectly.

8.2 Call Frame

Definition: boxes for parameters at the start of the function and for variables local to the function as they are created.

Deleted when call is done.

8.3 Call Stack

  • The set of function frames drawn in call order.
  • Functions frames are “stacked”: cannot remove one above without removing one below.
  • Python must keep the entire stack in memory: error if it cannot hold stack (“stack overflow”).
